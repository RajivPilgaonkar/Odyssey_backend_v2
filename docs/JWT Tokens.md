## JWT Token setting procedure

1. **`Node config files`**
* In the node terminal, type in `node`
* Then `require('crypto').randomBytes(64).toString('hex)`
* Copy the string to .env JWT_ACCESS_TOKEN_SECRET
* Then `require('crypto').randomBytes(64).toString('hex)` again
* Copy the string to .env 
JWT_ACCESS_REFRESH_TOKEN_SECRET
* But the secret key could be any string 


2. **`Dashboard (Verify Token)`**
* On Entry to the Dashboard screen, the token available in the Redux store is sent to the server for verification. The verification is based on the token and the expiration date
* If it is invalid, you are directed back to the Home Page for login
* By default, the token expires in a day, but this has been set in the AdmUsers table for every user 

3. **`Home (Generate Token)`**
* On the Home Page, every time the User Name & Password are entered, 
  and the Login button is clicked, a fresh token is generated by the server
* This fresh token is stored immediately to the redux store
* On moving to the Landing Page (Dashboard), this token is verified
